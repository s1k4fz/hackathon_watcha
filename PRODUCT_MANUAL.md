## 1. 产品愿景 (Product Vision)

**“拒绝冷冰冰的指令列表，让战斗回归角色扮演的本质。”**

传统的二次元 RPG 游戏（如《崩坏：星穹铁道》、《FGO》）虽然画面精美，但玩家与角色的互动仅限于点击固定的技能图标。**Project Soul Link** 旨在利用生成式 AI (GenAI) 打破“玩家”与“角色”之间的第四面墙，构建一个**自然语言驱动、即时语音反馈、高度个性化**的战斗体验。

我们的核心理念是：**指令即对话，人设即玩法。**

---

## 2. 核心亮点 (Key Highlights)

### 🌟 2.1 意图驱动的战术指挥系统 (Intent-Driven Command System)
- **告别机械操作**：对于预设角色（如琪亚娜、爱莉希雅），玩家不再需要点击“普通攻击”或“必杀技”，而是通过自然语言输入战术意图（例如：“琪亚娜，稍微试探一下对方！”或“别管那么多了，全力一击！”）。
- **AI 战术解析**：后台 LLM 充当“战术副官”，实时分析玩家的模糊指令，结合当前战场局势（HP、剩余敌人），自动决策最优技能释放，并生成符合角色性格的语音回复。
- **沉浸感拉满**：角色不仅是战斗单位，更是会思考、会吐槽、有情绪的伙伴。

### 🎭 2.2 零门槛 AI 角色生成 (Prompt-to-Playable Character)
- **一句话捏人**：玩家无需填写复杂的数值表，只需一段自然语言描述（如：“我是个怕痛的全点防御力的盾娘，喜欢吃甜食”），AI 即可在 3 秒内生成完整的可玩角色。
- **原子化技能组装**：系统将技能拆解为原子效果（Damage, Heal, Defense, Buff），AI 根据用户描述动态组装出独一无二的技能组和数值体系。
- **身份代入**：生成的角色直接作为玩家的“化身”进入战斗，拥有专属的战斗面板和技能按钮，实现“我”在游戏中的真实存在感。

### 🤝 2.3 上下文感知的双向互动 (Context-Aware Social Combat)
- **角色认识“你”**：预设角色（AI）能读取玩家自建角色的信息（姓名、性格）。在战斗对话中，琪亚娜可能会喊出你的名字，或者根据你的性格设定做出特定的反应（例如：“[玩家名]你还是这么冲动啊！”）。
- **动态语音 (TTS)**：集成了 Fish Audio 模型，实现了毫秒级的流式语音合成。琪亚娜、爱莉希雅等角色的回复不再是枯燥的文字，而是带有情绪的实时语音。

### 📊 2.4 硬核的战斗数值内核 (Solid Combat Mechanics)
- **崩铁级伤害公式**：尽管前端表现形式是 AI 驱动的，但底层战斗逻辑严谨可靠。实现了包含 **基础伤害区 × 增伤区 × 防御减免区 × 暴击区** 的完整伤害计算公式，并引入随机浮动值，保证了游戏性的深度。
- **可视化调试**：战斗日志实时展示详细的伤害计算过程，兼顾了娱乐性与硬核玩家的数据分析需求。

## 3. 用户体验流程 (User Journey)

1.  **初次进入**：
    *   系统引导玩家描述自己：“你在这个世界是谁？”
    *   AI 生成专属角色卡片（含属性、技能、性格），玩家确认后“灵魂同步”完成。

2.  **角色选择**：
    *   玩家组建小队，队伍中包含“我自己”（自建角色）和“伙伴”（预设 AI 角色，如琪亚娜）。

3.  **战斗开始**：
    *   **当轮到“伙伴”行动时**：
        *   玩家输入自然语言指令（语音/文字）。
        *   伙伴 AI 解析指令 -> 回复语音（带入玩家身份） -> 自动释放技能。
    *   **当轮到“我自己”行动时**：
        *   UI 切换为传统 RPG 技能面板（体现“我”对自己身体的完全掌控）。
        *   玩家直接点击技能 -> 释放技能 -> 触发战斗特效。

4.  **战斗结算**：
    *   基于战斗表现生成总结，甚至可以让 AI 角色对这场战斗进行复盘吐槽。

## 4. 技术架构 (Technical Architecture)

*   **前端框架**: React 18 + Vite + TypeScript (保证高性能与类型安全)
*   **UI/UX**: Tailwind CSS + Framer Motion (实现丝滑的动态交互与毛玻璃特效)
*   **AI 核心**:
    *   **大脑**: OpenAI/OpenRouter API (LLM) - 用于意图识别、角色扮演、数值生成。
    *   **嘴巴**: Fish Audio API - 用于高拟真、低延迟的角色语音合成。
    *   **耳朵 (Future)**: 浏览器 Web Speech API (语音转文字)。
*   **状态管理**: React Hooks (`useBattle`) 实现的复杂回合制状态机。

## 5. 未来展望 (Roadmap)

如果这是一个持续运营的产品，接下来的迭代方向：

1.  **多模态交互升级**:
    *   支持直接语音输入指令（彻底解放双手）。
    *   AI 生成角色立绘（接入 Stable Diffusion/Midjourney）。
2.  **更复杂的战斗机制**:
    *   引入“元素弱点击破”、“行动条 (AV值)”、“战技点”等机制。
    *   支持多对多 (Team vs Team) 战斗。
3.  **情感羁绊系统**:
    *   根据战斗中的配合程度，提升与 AI 角色的“信赖度”，解锁特殊语音或合体技。
4.  **Roguelike 模式**:
    *   玩家生成的角色在冒险中不断获得新的“原子词条”，进化技能。
